# BiliArm 浏览器插件设计文档

## 1. 项目概述

BiliArm 是一个面向哔哩哔哩网页播放体验优化的浏览器插件。插件通过统一的设置面板，为用户提供播放控制、弹幕控制、灯光模式、默认显示模式和页面元素清理等能力，让用户在观看视频时减少重复操作，并获得更稳定、沉浸、可控的播放体验。

插件的设置页面风格参考 [biliplus](https://github.com/0xlau/biliplus)：整体应保持简洁、清晰、偏工具型，重点突出功能开关、状态反馈和配置项分组。

## 2. 设计目标

- 提供一个全局总开关，允许用户快速启用或停用插件所有增强能力。
- 将每项功能设计为独立开关，用户可以按需开启，不强制改变所有观看习惯。
- 在 B 站视频播放页自动生效，尽量减少用户每次打开视频后的重复操作。
- 优先使用浏览器插件标准能力实现，避免侵入页面逻辑过深，降低后续维护成本。
- 设置界面清楚表达每项功能的作用，避免用户需要阅读复杂说明才能使用。

## 3. 用户场景

### 3.1 常规观看

用户打开 B 站视频后，插件根据配置自动关闭弹幕、自动播放、切换默认显示模式，用户无需重复点击播放器按钮。

### 3.2 宽屏或网页全屏观看

用户经常使用宽屏或网页全屏模式，希望页面背景变暗、底部弹幕不遮挡字幕，并且滚动页面时可以自动开关灯光模式。

### 3.3 快捷键控制

用户希望空格键始终用于播放或暂停，而不是在页面焦点不在播放器上时触发页面滚动。

### 3.4 页面清理

用户不希望首页或大屏模式下出现轮播图等干扰元素，希望插件自动移除或隐藏这些内容。

## 4. 功能范围

### 4.1 核心功能

| 功能 | 默认状态 | 说明 |
| --- | --- | --- |
| 插件总开关 | 开启 | 控制所有增强功能是否生效 |
| 启用快捷键 | 开启 | 开启后接管指定快捷键行为 |
| 空格键播放/暂停 | 开启 | 空格键始终控制视频播放或暂停，阻止页面滚动 |
| 默认关闭弹幕 | 关闭 | 进入播放页后自动关闭弹幕 |
| 自动播放 | 关闭 | 进入播放页后自动开始播放 |
| 播放结束退出全屏 | 关闭 | 视频播放结束后自动退出网页全屏或浏览器全屏 |
| 屏蔽底部弹幕 | 关闭 | 隐藏底部弹幕，减少遮挡字幕的情况 |
| 默认关灯 | 关闭 | 进入播放页后自动开启关灯/沉浸观看模式 |
| 滚动自动开关灯 | 关闭 | 页面上下滚动时自动开灯或关灯，适合宽屏观看 |
| 默认显示模式 | 正常 | 可选：正常、宽屏、网页全屏 |
| 去除大屏轮播图 | 关闭 | 自动隐藏或移除大屏轮播图 |

### 4.2 暂不包含

- 不处理移动端 App 或移动网页端。
- 不下载视频、音频或弹幕文件。
- 不绕过 B 站权限、会员、登录或地区限制。
- 不修改用户账号数据或服务端数据。
- 不提供复杂脚本市场或第三方脚本注入能力。

## 5. 插件形态

插件建议采用 Manifest V3 浏览器扩展架构，优先兼容 Chromium 系浏览器，后续可评估 Firefox 兼容。

### 5.1 主要组成

- `manifest.json`：声明插件元信息、权限、页面匹配规则和脚本入口。
- `background service worker`：处理插件生命周期、默认配置初始化和跨页面消息。
- `content script`：注入 B 站页面，负责识别播放器、绑定事件和执行页面增强逻辑。
- `options page`：完整设置页面，用于管理总开关和所有功能配置。
- `popup`：轻量快捷面板，用于快速切换总开关和常用功能。
- `storage`：保存用户配置，建议使用 `chrome.storage.sync`，方便跨浏览器同步。

### 5.2 页面匹配范围

建议优先支持以下页面：

- `https://www.bilibili.com/video/*`
- `https://www.bilibili.com/bangumi/play/*`
- `https://www.bilibili.com/list/*`
- `https://www.bilibili.com/`

其中播放控制类能力主要在视频和番剧播放页生效，页面清理类能力可在首页、大屏页或其他目标页面生效。

## 6. 配置模型

建议使用集中式配置对象，便于版本升级和默认值合并。

```json
{
  "enabled": true,
  "hotkeys": {
    "enabled": true,
    "spacePlayPause": true
  },
  "player": {
    "defaultDanmakuOff": false,
    "autoPlay": false,
    "exitFullscreenOnEnded": false,
    "defaultViewMode": "normal"
  },
  "danmaku": {
    "hideBottomDanmaku": false
  },
  "light": {
    "defaultLightsOff": false,
    "autoToggleOnScroll": false
  },
  "pageCleanup": {
    "removeLargeCarousel": false
  }
}
```

### 6.1 默认显示模式枚举

| 值 | 名称 | 说明 |
| --- | --- | --- |
| `normal` | 正常 | 使用 B 站默认播放器大小 |
| `wide` | 宽屏 | 自动切换到宽屏模式 |
| `webFullscreen` | 网页全屏 | 自动切换到网页全屏模式 |

## 7. 设置界面设计

### 7.1 信息架构

设置页面建议分为以下区域：

1. 顶部区域：插件名称、简短描述、总开关。
2. 快捷键设置：快捷键启用状态、空格键播放/暂停。
3. 播放设置：自动播放、默认显示模式、播放结束退出全屏。
4. 弹幕设置：默认关闭弹幕、屏蔽底部弹幕。
5. 灯光设置：默认关灯、滚动自动开关灯。
6. 页面清理：去除大屏轮播图。
7. 底部区域：版本号、重置默认设置、保存状态提示。

### 7.2 交互原则

- 总开关关闭时，其余配置项在视觉上置灰，但保留用户已选择的配置。
- 每个功能开关即时保存，无需额外点击“保存”按钮。
- 修改成功后展示轻量状态反馈，例如“已保存”。
- 默认显示模式使用单选按钮或分段控件，不建议使用自由输入。
- 对可能影响观看行为的功能提供简短说明，例如“滚动自动开关灯”。

### 7.3 视觉风格

- 页面整体使用浅色背景和卡片式分区，参考 biliplus 的简洁插件设置风格。
- 功能组之间保持明确间距，避免所有开关堆在一起。
- 开关控件使用统一样式，开启状态突出，关闭状态弱化。
- 文案保持短句，突出功能结果，而不是技术实现。

## 8. 核心模块设计

### 8.1 配置模块

职责：

- 读取默认配置。
- 从浏览器存储读取用户配置。
- 合并默认配置和历史配置。
- 监听配置变化并通知页面模块更新行为。

关键点：

- 配置需要支持版本迁移，避免后续新增字段导致老用户配置缺失。
- content script 不应假设配置一定完整，需要在读取时做默认值补齐。

### 8.2 页面识别模块

职责：

- 判断当前页面是否为支持的 B 站页面。
- 判断页面类型，例如视频页、番剧页、首页或大屏页。
- 等待播放器节点出现后再执行播放器相关逻辑。

关键点：

- B 站是动态页面，页面切换可能不刷新浏览器标签页。
- 需要监听 History API 或 DOM 变化，在路由变化后重新执行初始化。

### 8.3 播放器控制模块

职责：

- 自动播放。
- 默认关闭弹幕。
- 切换默认显示模式。
- 播放结束后退出全屏。

实现策略：

- 优先通过页面已有按钮触发行为，减少直接调用内部变量的依赖。
- 如果按钮状态可识别，应先判断当前状态，避免重复点击导致反向切换。
- 对播放器加载使用重试机制，避免页面尚未渲染完成时执行失败。

### 8.4 快捷键模块

职责：

- 在启用快捷键后监听 `keydown` 事件。
- 当用户按下空格键时，控制视频播放或暂停。
- 阻止空格键触发页面滚动。

约束：

- 当焦点位于输入框、文本域、可编辑元素时，不接管空格键。
- 当页面不存在可播放视频时，不阻止默认行为。
- 需要避免与 B 站原生快捷键产生明显冲突。

### 8.5 弹幕模块

职责：

- 默认关闭弹幕。
- 屏蔽底部弹幕。

实现策略：

- 默认关闭弹幕可通过识别弹幕开关状态并点击切换实现。
- 屏蔽底部弹幕优先通过播放器弹幕设置项实现；若页面不提供稳定入口，可通过 CSS 隐藏底部弹幕元素作为兜底方案。

### 8.6 灯光模块

职责：

- 进入播放页后默认开启关灯模式。
- 根据页面滚动位置自动开启或关闭关灯模式。

实现策略：

- 默认关灯应在播放器准备完成后执行。
- 滚动自动开关灯需要节流，避免频繁触发。
- 自动逻辑只在用户开启该功能时生效，且不能覆盖总开关关闭状态。

### 8.7 页面清理模块

职责：

- 去除大屏轮播图。
- 后续可扩展为隐藏其他干扰元素。

实现策略：

- 优先通过 CSS 注入隐藏目标元素。
- 对动态加载的轮播图使用 MutationObserver 监听并处理。
- 选择器需要集中维护，便于 B 站页面结构变化时更新。

## 9. 数据流

1. 插件安装或启动时初始化默认配置。
2. 用户在 popup 或 options page 修改配置。
3. 配置写入 `chrome.storage.sync`。
4. content script 监听配置变化。
5. 当总开关开启且当前页面匹配目标页面时，执行对应功能模块。
6. 当页面路由变化或播放器重新加载时，页面识别模块重新触发初始化。

## 10. 权限设计

建议最小化权限申请：

```json
{
  "permissions": ["storage"],
  "host_permissions": ["https://www.bilibili.com/*"]
}
```

如后续需要跨子域支持，可再增加对应 host permission。除非明确需要，不建议申请 `tabs`、`webRequest` 或更高风险权限。

## 11. 兼容性与风险

### 11.1 页面结构变化

B 站页面 DOM、类名和按钮结构可能变化，导致插件选择器失效。

应对方式：

- 将选择器集中定义。
- 对关键行为提供多套选择器兜底。
- 对找不到节点的情况静默失败，不影响页面正常使用。

### 11.2 与页面原生功能冲突

插件接管快捷键、弹幕和显示模式，可能与 B 站原生逻辑冲突。

应对方式：

- 只在用户明确开启功能时接管。
- 避免在输入场景中拦截按键。
- 执行前判断当前状态，减少重复切换。

### 11.3 动态路由问题

B 站部分页面切换不会刷新页面，content script 可能只执行一次。

应对方式：

- 监听 URL 变化。
- 使用 MutationObserver 等待播放器重新出现。
- 初始化逻辑需要幂等，避免重复绑定事件。

## 12. 测试方案

### 12.1 功能测试

- 总开关关闭后，所有增强功能不生效。
- 单独开启每个功能，确认只影响对应行为。
- 空格键在播放器页面控制播放/暂停。
- 空格键在评论框、搜索框、输入框中正常输入空格。
- 自动播放只在播放页执行，不影响非播放页。
- 默认关闭弹幕不会因重复执行而重新打开弹幕。
- 默认显示模式可正确切换正常、宽屏、网页全屏。
- 播放结束后可退出全屏或网页全屏。
- 去除大屏轮播图后页面布局不明显错乱。

### 12.2 兼容测试

- Chromium 浏览器最新版。
- Edge 浏览器最新版。
- 未登录 B 站状态。
- 已登录 B 站状态。
- 普通视频页。
- 番剧播放页。
- 页面内跳转到另一个视频。

### 12.3 回归测试

- 更新插件后保留旧配置。
- 新增配置项后老用户配置可以自动补齐默认值。
- 插件禁用或卸载后，B 站原页面行为恢复正常。

## 13. 开发里程碑

### 第一阶段：基础插件框架

- 创建 Manifest V3 插件结构。
- 实现配置默认值和存储读写。
- 实现 options page 和 popup 的基础 UI。
- 实现总开关和功能开关保存。

### 第二阶段：播放器增强

- 实现播放页识别。
- 实现空格键播放/暂停。
- 实现自动播放。
- 实现默认关闭弹幕。
- 实现默认显示模式。

### 第三阶段：沉浸观看能力

- 实现默认关灯。
- 实现滚动自动开关灯。
- 实现播放结束退出全屏。
- 实现屏蔽底部弹幕。

### 第四阶段：页面清理与稳定性

- 实现去除大屏轮播图。
- 增加动态路由监听。
- 增加 MutationObserver 兜底处理。
- 集中整理选择器和错误处理。

### 第五阶段：测试与发布准备

- 完成主要功能测试。
- 补充 README 和使用说明。
- 准备插件图标和版本信息。
- 打包本地安装包。

## 14. 后续扩展方向

- 支持按页面类型分别配置功能。
- 支持快捷键自定义。
- 支持配置导入和导出。
- 支持隐藏推荐视频、广告位或其他干扰元素。
- 支持深色设置页面。
- 支持 Firefox 扩展兼容。

## 15. 验收标准

- 用户可以在设置页面看到总开关和所有功能开关。
- 用户修改配置后刷新 B 站页面，配置仍然生效。
- 插件总开关关闭时，不改变 B 站页面播放行为。
- 开启空格键播放/暂停后，空格键不会导致播放页滚动。
- 开启默认关闭弹幕后，进入视频页后弹幕处于关闭状态。
- 开启默认显示模式后，进入视频页可自动切换到指定模式。
- 开启页面清理后，目标轮播图被隐藏或移除。
- 插件在目标页面静默运行，不在控制台持续输出无意义错误。
