# Better Bilibili 功能复刻清单

本文档记录 BiliArm 对 Better Bilibili 公开功能的复刻情况。功能来源参考 Chrome Web Store 与公开插件介绍页中列出的 Better Bilibili 能力，包括：微调布局、净化页面、评论 IP 属地显示、本地/账号黑名单、推荐过滤、优选播放链接、快捷拉黑和快捷操作等。

参考来源：

- Chrome Web Store：`https://chromewebstore.google.com/detail/better-bilibili/oofhgjpnnopfghkigjebdacghcoeanlh`
- OffiDocs 插件介绍页：`https://www.offidocs.com/bn/index.php/chromium-web-store/1783698-better-bilibili-extension?amp=1`

## 已复刻功能

| Better Bilibili 功能 | BiliArm 实现状态 | 说明 |
| --- | --- | --- |
| 微调布局 | 已实现 | 通过 CSS 优化首页和播放页局部布局稳定性。 |
| 净化页面 | 已实现 | 隐藏广告、活动横幅、推广位和播放结束推荐覆盖层。 |
| 评论 IP 属地显示 | 已实现 | 尽量恢复页面中已经渲染但被隐藏的 IP 属地元素。 |
| 隐藏置顶广告评论 | 已实现 | 按广告、推广、抽奖、活动等关键词隐藏疑似置顶广告评论。 |
| 显示视频话题标签 | 已实现 | 尽量恢复页面中已经存在的话题标签区域。 |
| 本地黑名单 | 已实现 | 设置页提供本地黑名单文本框，一行一个关键词或 UP 名。 |
| 本地黑名单过滤首页推荐 | 已实现 | 首页视频卡片命中黑名单后自动隐藏。 |
| 本地黑名单过滤播放页推荐 | 已实现 | 播放页右侧推荐卡片命中黑名单后自动隐藏。 |
| 首页悬停快速拉黑 | 已实现 | 鼠标悬停首页视频卡片时显示“本地拉黑”按钮。 |
| 播放页推荐四击拉黑 | 已实现 | 短时间内连续点击推荐卡片四次，将卡片文本加入本地黑名单。 |
| 首页点击空白三次回顶部 | 已实现 | 首页空白区域三击后滚动到顶部。 |
| 去除大屏轮播图 | 已实现 | content script 在 `document_start` 注入预隐藏样式，减少闪现。 |
| 换一换保留旧列表 | 已实现 | 首页点击换一换后，将原视频列表追加到新列表后方。 |

## 辅助实现或预留功能

| Better Bilibili 功能 | BiliArm 当前处理 | 说明 |
| --- | --- | --- |
| 账号黑名单 | 辅助实现待扩展 | 当前已实现本地黑名单。账号黑名单需要接入 B 站账号接口或原生拉黑流程，后续可在明确接口和权限后补齐。 |
| 黑名单导入导出 | 文本框式导入导出 | 设置页的本地黑名单文本框可直接粘贴、复制整份名单。后续可增加文件导入导出按钮。 |
| 优选视频播放链接 | 开关已预留 | 该能力需要在页面主世界 hook 播放请求或选择 CDN，当前普通 content script 无法稳定完成。 |
| 阻断追踪 WebSocket | 开关已预留 | 该能力需要主世界脚本或网络层拦截，后续单独实现。 |
| 打开设置面板快捷键 | 暂不启用 | 用户已要求隐藏快捷键相关功能，因此当前没有复刻该快捷键。 |

## 当前实现文件

- 配置模型：`src/common/config.js`
- 设置页面：`src/options/options.html`
- 设置交互：`src/options/options.js`
- 页面增强逻辑：`src/content/content.js`
- 样式：`src/common/settings.css`

## 后续建议

1. 使用 B 站真实页面继续校准选择器，尤其是评论 IP、话题标签和关灯按钮。
2. 如需完整复刻“优选播放链接”和“阻断追踪 WebSocket”，建议新增主世界脚本，并在文档中说明可能涉及的权限和风险。
3. 如需账号黑名单，建议优先确认是否调用 B 站公开接口、原生页面能力，或仅提供本地辅助拉黑。
